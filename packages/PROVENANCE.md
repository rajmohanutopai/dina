## Why this matters

1. **Reproducibility** — anyone can run `git -C dina-mobile archive 356ddc9 packages/core | tar -x …` and get the exact bytes this workspace started from.
2. **Licence attribution** — both repos are MIT; the move is an internal fork. Copyright notices in source files stay intact.
3. **Divergence audit** — if a bug appears that exists upstream in `dina-mobile`, `git blame` resolves to the donor commit for everything that hasn't been modified here since the import.
