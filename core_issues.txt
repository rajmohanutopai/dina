[CRITICAL-01] Persona state load failure is fail-open; can enable lock bypass after persona recreation
Evidence
- `core/cmd/dina-core/main.go:305-307`
  - If `persona_state.json` fails to load/parse, startup logs warning and continues with empty in-memory persona state.
- `core/internal/adapter/identity/identity.go:765-788`
  - `Create()` silently inserts/replaces persona metadata in memory and persists it.
- `core/internal/adapter/crypto/keyderiver.go:48-50`
  - Vault DEK is derived from seed + persona name + version, not from persona passphrase.
Impact
- If persona metadata is lost/corrupted but vault files remain, an operator with admin API access can recreate a persona with a new passphrase hash and regain vault access using the same derived DEK.
- This weakens the intended lock/unlock security boundary to availability of a single JSON metadata file.
Remediation
- In non-test/dev modes, fail startup on persona state load/unmarshal errors.
- Store persona lock/passphrase metadata in durable transactional storage (identity DB) with integrity checks, not a standalone best-effort JSON file.
- Add a recovery flow that requires explicit operator confirmation/migration rather than silent fresh-state fallback.

--- SUGGESTED SOLUTION ---
Agree. All three remediation points are valid.

1. `core/cmd/dina-core/main.go:305-307` — Replace the warn-and-continue with a hard failure
   in production mode:
   ```go
   if err := identSvc.LoadState(stateFile); err != nil {
       if os.Getenv("DINA_ENV") == "test" {
           log.Printf("WARN: persona state load failed (test mode, continuing): %v", err)
       } else {
           log.Fatalf("FATAL: persona state load failed: %v", err)
       }
   }
   ```

2. `core/internal/adapter/identity/identity.go` — Add an integrity anchor to persona
   metadata. When a persona is first created, compute and store
   `HMAC-SHA256(seed, persona_name)` as `dek_binding` alongside the persona record.
   On subsequent loads, verify the binding matches. This ensures that even if the
   JSON file is replaced, the DEK derivation path cannot be silently re-targeted:
   ```go
   type PersonaMeta struct {
       Name           string `json:"name"`
       Tier           string `json:"tier"`
       PassphraseHash string `json:"passphrase_hash"`
       DEKBinding     string `json:"dek_binding"` // HMAC(seed, name) — integrity anchor
       CreatedAt      string `json:"created_at"`
   }
   ```
   On `Create()`, compute and store the binding. On `LoadState()`, verify each
   persona's binding matches; reject any mismatched entries.

3. Add a `--recover-personas` CLI flag that explicitly drops and re-derives persona
   state, requiring operator confirmation, rather than doing it silently on every
   startup. Normal startup should refuse to proceed with corrupted state.
--- END SUGGESTED SOLUTION ---

- It also adds seed-dependent state coupling without preventing the core attack path.

- Keep step 1 (fail startup on state-load corruption in non-test mode).
- Add explicit create-time guard: if vault artifacts for a persona already exist but persona metadata is missing, reject create and require an explicit recovery/migration workflow.
- Long-term, move persona metadata to durable transactional identity storage with integrity checks.
--- END REVIEWER DISAGREEMENT ---

[HIGH-02] Persona deletion is incomplete (not persisted; dangling contacts/timers)
Evidence
- `core/internal/adapter/identity/identity.go:957-969`
  - `Delete()` removes from `personas` map only.
  - No `persistState()` call.
  - No cleanup of `contacts[persona]` or active TTL timer entries.
Impact
- Deleted personas can reappear after restart.
- Stale contact links and timer state can survive deletion.
Remediation
- In `Delete()`, remove persona, contacts entry, TTL timer entry, then persist state atomically.
- Add tests: delete + restart should not restore persona/contacts/timers.

--- SUGGESTED SOLUTION ---
Agree. The fix is straightforward — `Delete()` must clean up all associated state
and persist atomically.

`core/internal/adapter/identity/identity.go:957-969` — Expand `Delete()`:
```go
func (s *Service) Delete(name string) error {
    s.mu.Lock()
    defer s.mu.Unlock()

    if _, exists := s.personas[name]; !exists {
        return fmt.Errorf("persona %q not found", name)
    }

    // 1. Remove persona metadata
    delete(s.personas, name)

    // 2. Remove associated contacts
    delete(s.contacts, name)

    // 3. Cancel any active TTL timer
    if timer, ok := s.ttlTimers[name]; ok {
        timer.Stop()
        delete(s.ttlTimers, name)
    }

    // 4. Persist state atomically — if this fails, we have an inconsistency,
    //    so return the error and let the caller decide.
    if err := s.persistState(); err != nil {
        return fmt.Errorf("persona deleted in memory but persist failed: %w", err)
    }

    return nil
}
```

Tests to add:
- Delete persona → restart (re-load state) → persona must not exist.
- Delete persona → contacts for that persona must be gone.
- Delete persona → TTL timer for that persona must not fire.
--- END SUGGESTED SOLUTION ---

Partial disagreement on implementation details:

[HIGH-04] Dead-drop poison-pill DoS: invalid blobs are never evicted
Evidence
- `core/internal/ingress/sweeper.go:113-116` and `core/internal/ingress/sweeper.go:208-211`
  - On decrypt/verify failure, sweeper `continue`s without `Ack()`.
- `core/internal/ingress/sweeper.go:117-123` and `214-219`
  - TTL checks only run after successful decrypt/unmarshal.
Impact
- Malformed or hostile dead-drop blobs remain forever, consume spool capacity, and can permanently block new ingress once full.
Remediation
- Track per-blob retry count/first-seen time and evict or quarantine after threshold.
- Add size/age based garbage collection for undecodable blobs.
- Add tests for malformed blob lifecycle and capacity recovery.

--- SUGGESTED SOLUTION ---
Agree. The sweeper must bound the lifetime of unprocessable blobs.

`core/internal/ingress/sweeper.go` — Add retry tracking and eviction:

1. Add a failure tracker (in-memory map, persisted optionally):
   ```go
   type blobFailure struct {
       FirstSeen time.Time
       Count     int
   }

   type Sweeper struct {
       // ... existing fields ...
       failures map[string]*blobFailure // keyed by blob ID
       mu       sync.Mutex
       maxRetries   int           // default: 5
       maxAge       time.Duration // default: 24h
   }
   ```

2. In `Sweep()` and `SweepFull()`, on decrypt/verify failure:
   ```go
   case err != nil:
       s.mu.Lock()
       f, ok := s.failures[blobID]
       if !ok {
           f = &blobFailure{FirstSeen: time.Now()}
           s.failures[blobID] = f
       }
       f.Count++
       evict := f.Count >= s.maxRetries || time.Since(f.FirstSeen) > s.maxAge
       s.mu.Unlock()

       if evict {
           log.Printf("ingress: evicting poison blob %s (retries=%d, age=%s)",
               blobID, f.Count, time.Since(f.FirstSeen))
           _ = store.Ack(ctx, blobID) // remove from spool
           s.mu.Lock()
           delete(s.failures, blobID)
           s.mu.Unlock()
       }
       continue
   ```

3. Add periodic GC for the failures map itself (remove entries for blobs that
   were externally cleaned up).

Tests:
- Insert 10 malformed blobs → sweep N+1 times → all evicted, spool empty.
- Insert 1 malformed + 1 valid → sweep → valid processed, malformed tracked.
- Age-based eviction: insert blob, advance clock past maxAge, sweep → evicted.
--- END SUGGESTED SOLUTION ---

- The proposal does not explicitly call out restart semantics (in-memory poison tracking resets on process restart).
Recommended correction:
- Implement bounded invalid-blob eviction against current `deadDrop.List/Peek/Ack(name)` API.
- Document non-durable tracking semantics and ensure invalid blobs cannot permanently exhaust spool capacity.
--- END REVIEWER DISAGREEMENT ---

--- AUTHOR RESPONSE ---
Accepted. The reviewer is correct on both points. Verified against source:

1. **API naming mismatch**: In `sweeper.go`, the blob identifier variable is `name`
   (line 98: `for _, name := range blobs {`), not `blobID`. The Ack call is
   `s.deadDrop.Ack(name)` (e.g., lines 127, 173, 224, 275), not `store.Ack(ctx, blobID)`.
   `DeadDrop.Ack(name string) error` at `deaddrop.go:140` takes a single string
   param (no context) and does `os.Remove(filepath.Join(d.dir, name))`. My

   context parameter.

2. **Restart semantics**: The Sweeper struct (`sweeper.go:25-36`) has no persistent
   state — no database, no on-disk failure counters. The proposed in-memory

Revised approach using correct APIs:
```go
// In Sweep()/SweepFull(), on decrypt/verify failure:

f, ok := s.failures[name]
if !ok {
    f = &blobFailure{FirstSeen: time.Now()}

        name, f.Count, time.Since(f.FirstSeen))
    _ = s.deadDrop.Ack(name) // filesystem remove
    s.mu.Lock()
    delete(s.failures, name)
    s.mu.Unlock()
}
```

For restart resilience:
- Add a secondary **age-based GC sweep** that unconditionally evicts blobs older
  than `maxAge` based on filesystem mtime (available from `os.Stat`), regardless
  of in-memory retry count. This survives restarts because the eviction signal
  (file age) is durable.
- This means: in-memory tracking handles rapid-fire poison (evict after N retries
  within a session), and mtime-based GC handles long-lived poison across restarts
  (evict after 24h regardless).
- Document: "In-memory retry tracking resets on restart. Blobs surviving past
  `maxAge` are evicted by filesystem age check regardless of retry state."

Remediation

--- SUGGESTED SOLUTION ---
Agree. The `z` prefix in multibase means base58btc, not hex.

1. Add a `decodeMultibase()` helper in `core/internal/service/transport.go` (or a
   shared `core/internal/util/multibase.go`):
   ```go
   func decodeMultibase(encoded string) ([]byte, error) {
       if len(encoded) < 2 {
           return nil, fmt.Errorf("multibase string too short")
       }
       switch encoded[0] {
       case 'z': // base58btc
           return base58.Decode(encoded[1:])
       case 'f': // base16 lowercase
           return hex.DecodeString(encoded[1:])
       default:
           return nil, fmt.Errorf("unsupported multibase prefix: %c", encoded[0])
       }
   }
   ```

2. `core/internal/service/transport.go:136-145` and `445-452` — Replace

   Option (a) is preferred for W3C DID spec conformance.

4. Add a dependency on a base58 library (e.g., `github.com/mr-tron/base58`).

Tests:
- Round-trip: encode key as `z` + base58btc → decode → matches original bytes.
- Reject: `z` + invalid-base58 → error.
- Reject: unknown prefix → error.
--- END SUGGESTED SOLUTION ---

Accepted. The reviewer is correct on all three points. Verified against source:

1. `identity.go:129` defines `ed25519MulticodecPrefix = []byte{0xed, 0x01}` and

2. `transport.go:142` and `449` use `hex.DecodeString(multibaseKey[1:])` which is
   doubly wrong: wrong encoding (hex instead of base58btc) AND no multicodec
   prefix stripping. `device.go:114-119` already implements the correct pattern:

Revised `decodeMultibase()` helper:
```go
func decodeMultibase(encoded string) ([]byte, error) {
    if len(encoded) < 2 {
        return nil, fmt.Errorf("multibase string too short")
    }
    switch encoded[0] {
    case 'z': // base58btc
        raw, err := base58.Decode(encoded[1:])
        if err != nil {
            return nil, fmt.Errorf("base58btc decode: %w", err)
        }
        // Validate and strip Ed25519 multicodec prefix (0xed 0x01)

    default:
        return nil, fmt.Errorf("unsupported multibase prefix: %c", encoded[0])
    }
}
```

[HIGH-06] Outbound D2D endpoint is unvalidated (SSRF risk)
Evidence
- `core/internal/adapter/transport/transport.go:224-229`
  - `serviceEndpoint` from DID doc is accepted as-is.
- `core/internal/adapter/transport/transport.go:163-165`

Remediation

1. Add a `validateEndpoint()` function in `core/internal/adapter/transport/transport.go`:

   ```go
       u, err := url.Parse(rawURL)
       if err != nil {
           return fmt.Errorf("invalid endpoint URL: %w", err)
       }

       // Scheme check
       if u.Scheme != "https" {
           if os.Getenv("DINA_ENV") != "test" {
               return fmt.Errorf("only HTTPS endpoints allowed in production")
           }
           if u.Scheme != "http" {
               return fmt.Errorf("unsupported scheme: %s", u.Scheme)
           }
       }

       // Resolve hostname to IPs and check for private ranges
       host := u.Hostname()
       ips, err := net.LookupIP(host)
       if err != nil {
           return fmt.Errorf("cannot resolve endpoint host %q: %w", host, err)
       }
       for _, ip := range ips {
           if ip.IsLoopback() || ip.IsLinkLocalUnicast() || ip.IsLinkLocalMulticast() ||
               ip.IsPrivate() || ip.IsUnspecified() {
               return fmt.Errorf("endpoint resolves to denied IP range: %s", ip)
           }
       }

       return nil
   }
   ```
